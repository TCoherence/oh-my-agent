### Session 1D: Markdown-Aware Chunker

**Owner**: Codex
**Depends on**: nothing
**Estimated**: 0.5 session

#### Modify files

```
src/oh_my_agent/utils/chunker.py
tests/test_chunker.py (new)
```

#### Current behavior (39 lines)

Split priority: `\n\n` > `\n` > ` ` > hard cut.
No awareness of fenced code blocks.

#### New behavior

1. Before splitting, identify fenced code block ranges (` ``` ` or ` ~~~ `)
2. When choosing a split point, never split inside a fenced code block
3. If a single code block exceeds `max_size`, split inside the block by lines,
   closing the fence at split and re-opening in the next chunk:
   ```
   chunk 1: ... ```python\nline1\nline2\n```
   chunk 2: ```python\nline3\nline4\n``` ...
   ```
4. Preserve existing paragraph > newline > space priority for non-code-block text
5. Keep `MAX_CHUNK_SIZE = 1990` default

#### Tests

10 test cases:
- [ ] Plain text without code blocks (unchanged behavior)
- [ ] Text with code block that fits in one chunk (no split)
- [ ] Text where split point would fall inside code block → move split before/after block
- [ ] Code block exceeds max_size → split inside by lines, re-open fence
- [ ] Code block with language tag (```python) → preserved across splits
- [ ] Multiple code blocks in one message
- [ ] Code block at the very end of message
- [ ] Code block at the very start of message
- [ ] Nested backticks inside code block (``` inside ``` block)
- [ ] Empty code block

#### Acceptance

- [ ] Fenced code blocks are not corrupted during splitting
- [ ] Oversized code blocks are split with fence close/reopen
- [ ] All existing chunking behavior preserved for non-code text

#### Shutdown coordinator in main.py

```python
async def _shutdown(gateway_manager, scheduler, runtime_service, memory_store):
    """Coordinated shutdown sequence."""
    # 1. Stop scheduler (no new job dispatch)
    if scheduler:
        scheduler.stop()
    # 2. Stop gateway (reject new messages, drain in-flight)
    await gateway_manager.stop()
    # 3. Stop each channel
    for channel, _ in gateway_manager.channels:
        await channel.stop()
    # 4. Stop runtime (interrupt active subprocesses)
    if runtime_service:
        await runtime_service.stop()
    # 5. Close stores (flush WAL, close connections)
    if memory_store:
        memory_store.close()
```

#### Signal handling

```python
loop = asyncio.get_event_loop()
for sig in (signal.SIGINT, signal.SIGTERM):
    loop.add_signal_handler(sig, lambda: asyncio.create_task(_shutdown(...)))
```
