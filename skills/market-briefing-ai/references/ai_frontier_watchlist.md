## What belongs in the frontier radar

Use this section for:

- new or near-release frontier models
- official roadmap signals from labs
- major frontier capability claims
- deployment, access, pricing, or API moves that change the competitive picture
- high-signal rumors that are important enough to watch, but still clearly labeled if unverified

Examples of content that should land here by default:

- GPT-6 related signals
- Anthropic Mythos related signals
- major Gemini frontier updates
- new Qwen / DeepSeek flagship releases

## Source hierarchy

Use source priority in this order:

1. `official`
   - company blog
   - docs
   - model card
   - release note
   - API changelog
2. `high-quality media`
   - Reuters, Bloomberg, FT, WSJ, The Information, Semafor, etc.
3. `community / social`
   - X.com posts, leaks, screenshots, forum posts, newsletters

## Rumor discipline

- official sources can support main conclusions
- high-quality media can support secondary conclusions or triangulation
- unverified community/social signals must **not** become the report's mainline conclusion on their own
- if a rumor is not cross-checked, keep it in:
  - `unverified frontier signals`, or
  - watchlist notes
- do not use one rumor as the entire daily thesis

## Mainline vs watchlist decision rule

Promote a frontier signal into the mainline only when at least one of these is true:

- official confirmation exists
- two strong source families align
- the signal materially changes market positioning, ecosystem interpretation, or developer expectations

Otherwise keep it in watchlist / unverified form.
